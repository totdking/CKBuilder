# schemas

CKB uses [Molecule](https://github.com/nervosnetwork/molecule) — a compact binary serialization format — for all on-chain data. This folder contains the schema definition and the generated Rust types.

Molecule is used (instead of JSON or protobuf) because it is:
- **Deterministic** — identical data always produces identical bytes, which is required for consensus hashing 
- **Zero-copy** — fields can be read directly from the raw bytes without allocation
- **Schema-forward** — new optional fields can be added without breaking old parsers

---

## parameters.mol

The Molecule schema file. Defines the binary layout of all CKB transaction types:

| Type | Description |
|------|-------------|
| `Byte32` | 32-byte fixed array (tx hashes, code hashes) |
| `Uint32` / `Uint64` | Little-endian unsigned integers |
| `Script` | `{ code_hash, hash_type, args }` — a lock or type script |
| `ScriptOpt` | Optional `Script` (None for cells with no type script) |
| `OutPoint` | `{ tx_hash: Byte32, index: Uint32 }` — unique cell reference |
| `CellInput` | `{ since: Uint64, previous_output: OutPoint }` |
| `CellOutput` | `{ capacity: Uint64, lock: Script, type_: ScriptOpt }` |
| `CellDep` | `{ out_point: OutPoint, dep_type: Byte }` |
| `WitnessArgs` | `{ lock: BytesOpt, input_type: BytesOpt, output_type: BytesOpt }` |
| `RawTransaction` | The unsigned tx body (no witnesses) — what gets hashed for the tx id |
| `Transaction` | `RawTransaction` + `witnesses` — the complete signed transaction |

---

## parameters.rs

Auto-generated Rust types from the schema. Do not edit this file manually — it is produced by running:

```sh
moleculec --language rust --schema-file ./src/schemas/parameters.mol > ./src/schemas/parameters.rs
```

This generates builder types, accessor methods, and `as_slice()` for every Molecule table and struct defined in the schema. These are the types used throughout the network layer (e.g. `MolCellOutput`, `MolWitnessArgs`, `TransactionBuilder`).

---

## mod.rs

Re-exports all generated types under `crate::schemas` so they can be imported with a single `use crate::schemas::*` in the network layer.

---

## How Molecule is used in this project

The network layer (`transaction.rs`) has `pack()` methods on each Rust struct (`OutPoint`, `CellInput`, `CellOutput`, `CellDep`, `WitnessArgs`, `CkbScript`) that convert from the domain types into their Molecule counterparts. These are used when:

- Computing the **raw transaction hash** (`rpc_raw_tx_hash`) — the Molecule bytes are fed into Blake2b
- Computing the **sighash** (`create_rpc_sighash`) — witnesses are Molecule-packed before hashing
- **Broadcasting** via RPC — `to_rpc_value()` uses the packed witnesses as hex strings in the JSON payload
